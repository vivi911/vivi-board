## ⚠️ 工作規範（每次任務都必須遵守）

### 完成任務後必須做以下三件事：

1. 更新本檔案（CLAUDE.md），把改了什麼記錄進去
2. commit + push 到 GitHub
3. 輸出變更摘要，格式如下：

---
任務：（任務名稱）
動了哪些檔案：
  - 檔案名稱（說明改了什麼）
可能影響的功能：
測試方式：
有沒有推 git：✅ / ❌
---

### 需要 code review 時，額外輸出：

---
⚠️ Code Review 請確認：
- [ ] 有沒有 hardcode 的 API key 或敏感資料
- [ ] 有沒有 error handling
- [ ] 邏輯有沒有明顯漏洞
- [ ] 有沒有應該更新但沒更新的 CLAUDE.md
---

### 任務切割規則
- 每個任務必須能在 3 分鐘內完成
- 如果任務太大，主動拆分成多個小任務，一次只做一件事
- 範例：
  ❌ 「升級知識圖譜」
  ✅ 「任務一：建立知識圖譜 API」
  ✅ 「任務二：建立前端視覺化」
  ✅ 「任務三：整合 Firestore」

### 任務失敗處理規則
- 任務失敗時，不能靜默消失
- 必須輸出失敗原因
- 必須說明建議的下一步
- 格式如下：

---
❌ 任務失敗
失敗原因：（說明）
影響範圍：（哪些功能受影響）
建議下一步：（怎麼拆小或怎麼修）
---
